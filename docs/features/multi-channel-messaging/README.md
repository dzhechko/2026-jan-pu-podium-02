# Multi-Channel Messaging / Многоканальная рассылка

## Обзор

ReviewHub поддерживает отправку запросов на отзыв через три канала:
- **SMS** (SMSC.ru) — основной канал, работает всегда
- **Telegram** — через Telegram Bot API
- **Max** (ex-OK Messenger) — через Max Bot API

## Как работает отправка

### Архитектура

```
Админ выбирает канал → MessageGateway → Provider (Telegram/Max/SMS)
                                      ↓ (при ошибке)
                                      SMS fallback (автоматически)
```

1. Админ выбирает канал отправки на странице "Клиенты"
2. Система пытается отправить через выбранный канал
3. Если мессенджер недоступен — автоматический fallback на SMS
4. При fallback шаблон сообщения перезагружается (SMS-формат вместо Markdown)

### Сообщения идут напрямую клиенту

Бот отправляет сообщения **в личный чат с клиентом**, НЕ в группы. Это работает как личное сообщение от бота.

**Важно:** клиент должен **первым написать боту** (нажать /start в Telegram или написать боту в Max), иначе бот не имеет права отправлять сообщения.

## Настройка Telegram

### 1. Создание бота

1. Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Введите имя бота (например, "ReviewHub Отзывы")
4. Введите username бота (например, `reviewhub_reviews_bot`)
5. BotFather выдаст токен вида: `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`

### 2. Настройка в ReviewHub

1. Перейдите в **Настройки** → **Каналы доставки**
2. Вставьте токен в поле "Telegram Bot Token"
3. Нажмите "Проверить" — система проверит токен и покажет username бота
4. Сохраните настройки

### 3. Подключение клиентов

Клиент должен:
1. Найти вашего бота в Telegram (по ссылке `t.me/ваш_бот`)
2. Нажать **Start** (или отправить `/start`)
3. Узнать свой Chat ID через [@userinfobot](https://t.me/userinfobot) — бот покажет числовой ID

Вы вводите Chat ID при создании клиента или через CSV-импорт.

## Настройка Max

### 1. Создание бота

1. Перейдите на [botapi.max.ru](https://botapi.max.ru)
2. Создайте нового бота
3. Скопируйте токен из настроек бота

### 2. Настройка в ReviewHub

1. Перейдите в **Настройки** → **Каналы доставки**
2. Вставьте токен в поле "Max Bot Token"
3. Нажмите "Проверить" — система проверит токен и покажет имя бота
4. Сохраните настройки

### 3. Подключение клиентов

Клиент должен:
1. Найти вашего бота в Max и написать ему
2. Chat ID можно получить из webhook-событий бота (при получении сообщения от клиента)

Вы вводите Chat ID при создании клиента или через CSV-импорт.

## CSV-импорт с каналами

Формат CSV расширен для поддержки каналов:

```csv
name,phone,email,telegram_chat_id,max_chat_id,preferred_channel
Иван Иванов,+79001234567,ivan@mail.ru,123456789,,telegram
Мария Петрова,+79001234568,,,,sms
Алексей Сидоров,+79001234569,,,chat_abc123,max
```

- `telegram_chat_id` — числовой ID (только цифры)
- `max_chat_id` — строковый ID чата в Max
- `preferred_channel` — `sms`, `telegram`, или `max` (auto-detect если не указан)

## Шаблоны сообщений

Каждый канал использует свой формат:

| Канал | Формат | Особенности |
|-------|--------|-------------|
| SMS | Текст | Краткий, ссылки как URL |
| Telegram | Markdown | Жирный текст, emoji, inline-ссылки |
| Max | Markdown | Аналогично Telegram |

Шаблоны настраиваются отдельно для каждого канала и каждого напоминания (0-4).

## Безопасность

- Bot-токены шифруются при хранении (AES-256-GCM)
- Chat ID клиентов шифруются при хранении
- Токены передаются в HTTP-заголовках, НЕ в URL
- Все данные хранятся на российском VPS (152-ФЗ)

## FAQ

**Q: Что будет если клиент заблокирует бота?**
A: Отправка через мессенджер не удастся, сообщение автоматически уйдёт по SMS.

**Q: Можно ли отправлять в группы?**
A: Нет, только в личные чаты. Бот работает 1-на-1 с клиентом.

**Q: Как автоматизировать получение Chat ID?**
A: В будущем можно настроить webhook бота — при нажатии /start Chat ID будет сохраняться автоматически.

**Q: Нужен ли отдельный бот для каждого администратора?**
A: Да, каждый администратор настраивает свои токены ботов в настройках. Это позволяет каждому бизнесу иметь бота со своим именем.
