# Пользовательские сценарии ReviewHub

## Сценарии администратора

### Сценарий 1: Первоначальная настройка

```
Админ                          Система
  │                               │
  ├─ Открывает /register ────────▶│
  │                               │
  ├─ Заполняет: email, пароль,   │
  │  название, телефон ──────────▶│
  │                               ├─ Создаёт аккаунт
  │                               ├─ Генерирует JWT
  │◀── Redirect на / ────────────┤
  │                               │
  ├─ Переходит в /settings ──────▶│
  │                               │
  ├─ Вводит ссылку Яндекс Карт ─▶│
  │                               ├─ Извлекает org_id
  ├─ Устанавливает скидку 15% ──▶│
  │                               ├─ Сохраняет настройки
  │◀── «Сохранено» ──────────────┤
  │                               │
```

### Сценарий 2: Добавление клиентов и отправка SMS

```
Админ                          Система
  │                               │
  ├─ Открывает /clients ─────────▶│
  │                               │
  ├─ Нажимает «Добавить» ───────▶│
  ├─ Имя: Иван, Тел: +790... ──▶│
  │                               ├─ Шифрует телефон (AES-256)
  │                               ├─ Сохраняет в БД
  │◀── Клиент добавлен ──────────┤
  │                               │
  ├─ Нажимает «SMS» у Ивана ────▶│
  │                               ├─ Создаёт review_request
  │                               ├─ Генерирует уникальный токен
  │                               ├─ Расшифровывает телефон
  │                               ├─ Отправляет SMS через SMSC.ru
  │                               ├─ Записывает sms_log
  │◀── «SMS отправлено» ─────────┤
  │                               │
```

### Сценарий 3: CSV-импорт

```
Админ                          Система
  │                               │
  ├─ Нажимает «Импорт CSV» ─────▶│
  ├─ Загружает файл (≤10 МБ) ──▶│
  │                               ├─ Парсит CSV
  │                               ├─ Для каждой строки:
  │                               │   ├─ Валидация телефона
  │                               │   ├─ Шифрование
  │                               │   └─ Вставка в БД
  │◀── Импортировано: 48 ─────────┤
  │    Пропущено: 2 (дубликаты)   │
  │    Ошибки: 1 (неверный тел.)  │
  │                               │
```

### Сценарий 4: Просмотр аналитики

```
Админ                          Система
  │                               │
  ├─ Открывает / (дашборд) ─────▶│
  │                               ├─ 6 запросов в параллель:
  │                               │   ├─ COUNT(sms_logs)
  │                               │   ├─ COUNT(reviews)
  │                               │   ├─ COUNT(positive)
  │                               │   ├─ COUNT(negative)
  │                               │   ├─ AVG(stars)
  │                               │   └─ GROUP BY date
  │◀── Дашборд с метриками ──────┤
  │                               │
  ├─ Выбирает период «7 дней» ──▶│
  │                               ├─ Пересчёт за 7 дней
  │◀── Обновлённые данные ────────┤
  │                               │
```

## Сценарии клиента

### Сценарий 5: Оставление положительного отзыва

```
Клиент                         Система
  │                               │
  │◀── SMS: "Оставьте отзыв" ────┤
  │                               │
  ├─ Открывает ссылку ──────────▶│
  │                               ├─ GET /api/reviews/form/:token
  │                               ├─ Проверяет: токен валиден?
  │                               │   не истёк? не использован?
  │◀── Форма: название, скидка ──┤
  │                               │
  ├─ Ставит ★★★★★ ──────────────▶│
  ├─ Пишет «Супер!» ────────────▶│
  ├─ Нажимает «Отправить» ──────▶│
  │                               ├─ POST /api/reviews/submit/:token
  │                               ├─ Сохраняет отзыв
  │                               ├─ Claude API → sentiment: POSITIVE
  │                               ├─ confidence: 0.95 (≥ 0.7)
  │                               ├─ routed_to: YANDEX_REDIRECT
  │                               ├─ Статус → REVIEWED
  │◀── redirect_url Яндекс Карт ─┤
  │                               │
  ├─ Нажимает «Открыть Яндекс» ─▶│ (внешняя ссылка)
  │                               │
```

```
Клиент                         Система
  │                               │
  ├─ Открывает ссылку ──────────▶│
  │◀── Форма ────────────────────┤
  │                               │

  │                               │

  │   RH-A1B2C3D4, скидка 15%»   │
  │                               │
```

### Сценарий 7: Отписка от SMS


### Сценарий 10: Fallback при недоступности LLM

```
Система
  │
  ├─ Отзыв получен
  ├─ Claude API → ошибка/таймаут
  ├─ Fallback: используем звёзды
  │   ├─ ≥ 4 звезды → POSITIVE
  │   └─ < 4 звезды → NEGATIVE
  ├─ confidence = 0.5 (fallback)
  ├─ Логируем ошибку API
  │
```
