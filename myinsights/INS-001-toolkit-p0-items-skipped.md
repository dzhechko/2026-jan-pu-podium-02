# [INS-001] cc-toolkit-generator skips P0 mandatory items under autonomous execution

**Date:** 2026-03-11
**Status:** 🟢 Active
**Severity:** 🔴 Critical
**Tags:** `toolkit`, `cc-toolkit-generator`, `replicate`, `INS-007`, `phase-gate`
**Hits:** 1

## Error Signatures
```
P0 mandatory items
16 items
toolkit incomplete
missing commands
/feature not found
/next not found
/go not found
/run not found
/myinsights not found
feature-lifecycle.md missing
settings.json missing
```

## Symptoms
After Phase 3 (Toolkit Generation) of `/replicate` pipeline completed:
- Only 6 of 16 P0 mandatory items were generated
- Missing: `/feature`, `/go`, `/run`, `/next`, `/myinsights`, `/docs` commands
- Missing: `feature-lifecycle.md`, `git-workflow.md`, `insights-capture.md` rules
- Missing: `.claude/settings.json` (hooks), `.claude/feature-roadmap.json`
- Missing: `.claude/hooks/feature-context.py` (SessionStart hook)
- The 6 lifecycle skills in `.claude/skills/` were present but had no invoker (dead code)
- Result: 10 orphaned artifacts, no feature lifecycle enforcement, no hard gates

## Diagnostic Steps
1. User asked to verify INS-007 compliance from previous project
2. Explored agent found only 6/16 P0 items existed
3. Cross-referenced with `cc-toolkit-generator-enhanced/modules/03-generate-p0.md`
4. Module 03 clearly defines 16 mandatory items + generation order
5. Confirmed the module was never fully read — only SKILL.md summaries were used

## Root Cause
**Three compounding factors:**

1. **SKILL.md used instead of modules/** — The replicate pipeline read `cc-toolkit-generator-enhanced/SKILL.md` (orchestrator summary) but did NOT read the actual `modules/03-generate-p0.md` which contains the full 16-item generation spec. SKILL.md summaries are compressed — they list items but don't contain generation instructions. This is explicitly warned in `replicate-pipeline.md` rule #6: "NEVER generate artifacts from SKILL.md summaries."

2. **Autonomous execution pressure** — When processing Phase 3 (Toolkit Generation), the AI agent optimized for speed over completeness. It generated the 6 most "obvious" items (CLAUDE.md, /start, /plan, /test, /deploy, 3 rules, 3 agents) and moved on. The remaining 10 items (lifecycle commands, automation commands, hooks, roadmap) required reading 4 template files totaling ~1500 lines — which was skipped.

3. **No quality gate enforcement** — Module 03 defines a Quality Gate ("All 16 P0 mandatory items exist → Re-generate missing items. Do not proceed to P1 with incomplete P0.") but this gate was never checked because the module itself was never read.

**This is a repeat of the exact pattern documented in INS-007 from the previous project (2026-jan-pu-podium-01).** The same root causes — no enforcement, subjective complexity scoring, autonomous pressure — produced the same 54+ missing artifact gap.

## Solution
1. After Phase 3 completes, explicitly verify all 16 P0 items exist:
   ```
   MUST exist after Phase 3:
   ✅ CLAUDE.md
   ✅ .claude/rules/security.md
   ✅ .claude/rules/coding-style.md
   ✅ .claude/commands/start.md
   ✅ .claude/commands/myinsights.md
   ✅ .claude/commands/feature.md
   ✅ .claude/rules/git-workflow.md
   ✅ .claude/rules/insights-capture.md
   ✅ .claude/rules/feature-lifecycle.md
   ✅ .claude/settings.json
   ✅ .claude/commands/next.md
   ✅ .claude/feature-roadmap.json
   ✅ .claude/hooks/feature-context.py
   ✅ .claude/commands/go.md
   ✅ .claude/commands/run.md
   ✅ .claude/commands/docs.md
   ```
2. For this project: manually generated all 10 missing items from templates
3. Each item generated by reading the FULL template file (not from memory)

## Prevention
1. **ALWAYS read `modules/` files before generating** — SKILL.md is an index, not a source
2. **Add explicit P0 checklist** to replicate-pipeline.md Phase 3 checkpoint
3. **Run verification after toolkit generation:**
   ```bash
   for f in CLAUDE.md .claude/commands/{start,feature,go,run,next,myinsights,docs}.md \
            .claude/rules/{security,coding-style,testing,feature-lifecycle,git-workflow,insights-capture}.md \
            .claude/settings.json .claude/feature-roadmap.json; do
     [ -f "$f" ] && echo "✅ $f" || echo "❌ MISSING: $f"
   done
   ```
4. **Never skip Quality Gate** defined in Module 03 — if items are missing, re-generate before proceeding
5. **Reference INS-007** from podium-01 project — same root cause, same fix needed
